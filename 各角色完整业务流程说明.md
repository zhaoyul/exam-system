# 各角色完整业务流程说明

整理时间：2026-05-24

## 一、说明口径

本说明按“角色 → 前端入口 → 后端接口 → 当前闭环情况”整理，便于直接验收。

状态口径：

- **完整度较高**：入口、主要接口、主操作链基本齐备
- **部分闭环**：有页面和接口，但仍缺关键规则或关键环节
- **演示态**：主要用于门户展示或静态/模拟数据承载

---

## 二、集团管理员

### 1. 主要入口

- 工作台：`/wb/cert`
- 认定机构：`/cert/organizations`
- 认定监督：`/cert/supervision`
- 认定统计：`/cert/statistics`
- 集团证书：`/certificate/issue`、`/cert/certificates-group`
- 历次认定：`/cert/historical`
- 批复设置：`/cert/approval-settings`
- 认定批复：`/cert/approval`
- 证书上报：`/cert/cert-report`
- 预警违规：`/cert/violations`
- 特办申请：`/cert/special`

### 2. 对应接口

- `/api/certification/organizations`
- `/api/certification/supervision`
- `/api/certification/statistics`
- `/api/certification/approvals`
- `/api/certification/approval-settings`
- `/api/certification/certificate-reports`
- `/api/certification/violations`
- `/api/certification/special-applications`
- `/api/certificate/issue`
- `/api/certification/certificates-group`

### 3. 完整业务流程

1. 登录系统，进入集团工作台查看待办、预警、统计
2. 查看分支机构认定计划、监督数据、违规预警
3. 对认定计划或申报事项进行批复处理
4. 在集团证书模块统一生成证书、查看制证状态
5. 在证书打印模块统一安排打印任务
6. 在证书上报模块办理备案/上报
7. 在历次认定、溯源相关页面复核历史批次和异常情况

### 4. 当前判断

- **完整度：部分闭环**
- 原因：
  - 集团端入口最完整
  - 但证书生成、证书编号、上报回执、违规规则、多证预警等仍缺少真实后端规则支撑

- 机构端认定流程：`/cert/exec/*`

### 2. 对应接口

- `/api/certification/execution/exam-rooms`
- `/api/certification/execution/registration-orgs`
- `/api/certification/execution/exam-staff`
- `/api/certification/execution/supervisors`
- `/api/certification/plans`
- `/api/certification/exam-registration`
- `/api/certification/exam-arrangement`
- `/api/certification/exam-session`
- `/api/score/entry`
- `/api/certification/score-publicity-manage`

### 3. 完整业务流程

#### 3.1 基础数据准备

1. 维护考场信息
2. 维护报名机构
3. 维护考务人员
4. 维护监考人员
5. 配置费用标准

#### 3.2 认定主流程

1. 制定计划：`/cert/exec/plans`
2. 考试报名：`/cert/exec/registration`
3. 考场编排：`/cert/exec/arrangement`
4. 考务安排：`/cert/exec/session`
5. 成绩管理：`/cert/exec/score`
6. 成绩公示：`/cert/exec/publicity`

> 当前页面实现中，机构管理员只走到前六步；证书管理与证书打印由集团端统一处理。

### 4. 当前判断

- **完整度：部分闭环**
- 原因：
  - 页面路径和流程容器已完整铺开
  - 但批量导入、照片校验、编排算法、推卷、状态推进、已办待办切换等仍未形成真实闭环

## 十、验收结论

可以按下面的说法验收本轮成果：

> 当前仓库已经把各角色的主要入口、菜单与页面骨架铺开，集团管理员、机构管理员、评价专家、督导人员、考务人员、监考人员、考生等角色都已有对应门户和页面；同时，后端 Clojure API 也已完成大面积映射。但从“完整业务流程”标准看，系统目前仍以页面承载和通用资源接口为主，真实闭环主要停留在认证、备案、个人中心、部分督导模块，等级认定主流程距离文档要求的规则化、可追溯、可导入、可上报、可打印的完整闭环仍有明显差距。
